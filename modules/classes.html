API Reference¶

This is the class and function reference for Bear. Please refer to the full user guide for further details, as the class and function raw specifications may not be enough to give full guidelines on their uses.

bear.core: Core Bear functionality¶

The bear.core sub-module defines the core functionality and templating behavior of the package.

Core package creation¶

There’s only one public method in the core submodule.

core.make_package(header, bear_location, …) Build the package.

bear.utils: Utilities¶

Utilities, IO and validation functions used across the package.

IO utilities¶

Utilities for reading/copying templates and formatting them.

utils.copy_to(copy_from_file, write_to_dir) Copy a template file elsewhere.
utils.read_write(read_from_file, write_to_dir) Read a template and write it as a package file.

Validation functions¶

utils.validate_args(args) Get the validated args.
utils.validate_license(license) Validate the license.
utils.validate_path(path, name) Validate the location where the path will be written.
utils.validate_project_name(name) Validate the project name.
utils.validate_requirements(req, cython_required) Validate the package requirements.

YAML-parsing functions¶

utils.parse_yaml(path, defaults) Parse the arguments from a YAML file.
